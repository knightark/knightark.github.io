查看 ASP.NET Core 标识体系结构

ASP.NET Core 标识是一个成员身份系统，可将用户注册和登录功能添加到 ASP.NET Core Web UI。 成员身份系统处理身份验证和授权问题。 身份验证涉及你的身份。 授权涉及允许你进行的操作。 因此，身份验证是授权的先决条件。

作为本地帐户创建的替代方案，标识支持 Facebook 和 Twitter 等外部登录提供程序。 使用基于 Cookie 的身份验证保存用户会话。 默认情况下，在登录时会创建 Cookie，并在注销时销毁。

使用所选的数据存储和数据访问技术保存成员身份数据。 默认数据访问技术是称为 Entity Framework (EF) Core 的对象关系映射程序 (O/RM)。 默认数据存储是 SQL Server。

EF Core 简化了与基础数据库的交互。 因此，EF Core 通常可以通过标识使用任何数据库提供程序。 数据库提供程序可用于 PostgreSQL、SQLite 和其他几种数据存储。 但是，PostgreSQL 是第三方提供商，因此没有资格获得 Microsoft 的支持。 标识还使你可以灵活使用所选的数据访问技术。 Dapper 是一种常用的替代选项。

  • 数据库提供程序是一种特定于数据库的库，它接受来自 EF Core 提供程序（未示出）的 SQL 并执行它。

查看项目要求

你的利益干系人已定义了以下业务要求：

  • 系统有两类经过身份验证的用户：员工和管理员。
  • 不允许匿名用户查看产品目录。
  • 员工只能查看产品目录。
  • 管理员可以修改产品。
  • 成功登录后，用户的名字和姓氏应显示在应用的标头中。

你的开发团队做出以下技术决策：

  • 标识数据应隔离在其自己的数据库中。
  • 支持标识的数据库表应属于默认架构。
  • 管理员将使用一次性令牌自行注册。
  • 应用必须支持使用 TOTP 身份验证器应用通过多重身份验证登录。
  • 数据库凭据应存储在 Azure Key Vault 中。
